Road Network Traversals Using Contraction Hierarchies

Road Networks are highly dense networks when arranged in graphs as nodes and edges, and can lead to very high computations when finding the shortest path between 2 nodes using standard algorithms like Djikstra's. Contraction Hierarchies are built on the idea that to travel between destinations, we first move from streets to highways and bigger highways and then move back down again. This proves to be thousands of times faster than Djikstra and therefore more effective in real-time applications.


credit: https://www.mjt.me.uk/posts/contraction-hierarchies/

Contraction Hierarchies use augmented graphs to compute the shortest paths. This consists of adding shortcuts by contracting less important nodes and spreading important nodes across the graph, while minimizing the number of edges. The node ordering plays a crucial role in preprocessing and query time. To obtain an optimal node ordering, a measure of importance is added to each node using hueristics and its order is maintained in a priority queue.

The importance criteria are governed by:

- Edge Difference [ed(v)] = Added Shortcuts - Incoming Degree - Outgoing Degree
- Number of Contracted Neighbours [cn(v)]
- Shortest Cover [sc(v)] = number of neighbours w of v that we have to shortcut to or from w after contracting v

Node Level [L(v)] = Upper bound on the edges in the shortest path from any node s to v in the augmented graph
Importance I(v) = a*ed(v) + b*cn(v) + c*sc(v) + d*L(v)

By adjusting the weights of the above parameters, we can find the effect on preprocessing and query time to find the shortest path.
