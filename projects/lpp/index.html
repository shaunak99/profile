Localization And Path Planning For Autonomous Vehicles

Using the ROS framework, we had created a basic prototype for simulating autonomous navigation. We used Gazebo to simulate our robot and performed Mapping, Localization and Path planning.

Gmapping
We first created a map of the environment using gmapping. We map the surrounding obstacles by using laser data from the depth sensor mounted on the robot.

Localization
Once we created the map, we used the robot_localization Package in ROS to perform state estimation. We fused sensor data, such as wheel encoders and IMU, to produce a local pose estimate, and similarly combined AMCL (Adaptive Monte Carlo Localalization) and gmapping to obtain a global pose estimate. The state estimates are combined using the ekf_localization node, which is an implementation of the Extended Kalman Filter.

Path Planning - Move Base
To perform path planning, we used move_base from the navigation stack in ROS, which takes the transform (tf) and odometry (odom) data from the the ekf_localization_nodes as inputs and performs local and global path planning.
